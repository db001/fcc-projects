Introduction

Angular is a Javascript framework that extends HTML in order to make creating dynamic websites quicker and easier. The latest version is Angular 4, and is smaller and faster than it's predecessors. Whilst the original angular was written in plain JavaScript, the following iterations use TypeScript in order to support developer productivity with tooling. However while starting out Angular code can be written plain JavaScript.

Angular is one of the components of the MEAN stack but can be used with other libraries and all the features can be modified to suit the needs of the application being developed.

This document is small sample from angular.io, for a more detailed look at Angular, it's recommended to go there.

Thing you should know

  • Before starting Angular you should have a good working knowledge of HTML and JavaScript
  • An understanding of the Internet and the basics of web development
  • Previous experience of TypeScript isn't necessary to begin, but will be useful when developing more complex apps.
  • How to set up a development environment using Node and NPM
  • Knowledge of Git would be useful for following the tutorials on the Angular main site.

Setting up for Angular

It's highly recommended that you develop locally on your machine. There are plenty of online IDEs that can give you a taste, but locally is the best way.

Make sure you have Node and npm installed. Then...

  1. Create a project folder.
  2. Clone the QuickStart seed into your project folder.
  3. Install the npm packages.
  4. Run npm start to launch the sample application.
How to clone the repository

Use these terminal commands to clone the quickstart seed:

git clone https://github.com/angular/quickstart.git quickstart
cd quickstart
npm install
npm start

The Basics

Angular is a framework for building client applications in HTML and either JavaScript or a language like TypeScript that compiles to JavaScript.

The framework consists of several libraries, sone core and some optional

Angular applications are made up of HTML templates composed with components and services. A component is a combination of an HTML template and a component class that controls a portion of the screen. These components contain Angularized markup. Application logic is added in services and and this all gets boxed up as a module.

Below is an example component that displays a simple string

import { Component } from '@angular/core';

@Component({
  selector: 'my-app',
  template: `<h1>Hello {{name}}</h1>`
})

export class AppComponent { name = 'Angular'; }

The @component is a decorator function that essentially describes how the HTML template and component clas work together.

The selector property tells Angular to display the component inside a custom my-app tag inside the HTML. 

<my-app>Loading AppComponent content here ...</my-app>

Angular Modules

Angular apps are modular and Angular has its own modularity system called Angular modules or NgModules.

Every Angular app has at least one Angular module class, the root module, conventionally named AppModule.

While the root module may be the only module in a small application, most apps have many more feature modules, each a cohesive block of code dedicated to an application domain, a workflow, or a closely related set of capabilities.

An Angular module, whether a root or feature, is a class with an @NgModule decorator.

Data Binding

Without a framework, you would be responsible for pushing data values into the HTML controls and turning user responses into actions and value updates. Writing such push/pull logic by hand is tedious, error-prone, and a nightmare to read as any experienced jQuery programmer can attest.

Angular Libraries

Angular ships as a collection of JavaScript modules. You can think of them as library modules.

Each Angular library name begins with the @angular prefix.

You install them with the npm package manager and import parts of them with JavaScript import statements.

For example, import Angular's Component decorator from the @angular/core library like this:

import { Component } from '@angular/core';

You also import Angular modules from Angular libraries using JavaScript import statements:

import { BrowserModule } from '@angular/platform-browser';

Components

A component controls a patch of screen called a view.

You define a component's application logic—what it does to support the view—inside a class. The class interacts with the view through an API of properties and methods.

For example, this HeroListComponent has a heroes property that returns an array of heroes that it acquires from a service. HeroListComponent also has a selectHero() method that sets a selectedHero property when the user clicks to choose a hero from that list.

export class HeroListComponent implements OnInit {
  heroes: Hero[];
  selectedHero: Hero;

  constructor(private service: HeroService) { }

  ngOnInit() {
    this.heroes = this.service.getHeroes();
  }

  selectHero(hero: Hero) { this.selectedHero = hero; }
}

Angular creates, updates, and destroys components as the user moves through the application. Your app can take action at each moment in this lifecycle through optional lifecycle hooks, like ngOnInit() declared above.

Templates

You define a component's view with its companion template. A template is a form of HTML that tells Angular how to render the component.

A template looks like regular HTML, except for a few differences. Below is an example template:

<h2>Hero List</h2>
<p><i>Pick a hero from the list</i></p>
<ul>
  <li *ngFor="let hero of heroes" (click)="selectHero(hero)">
    {{hero.name}}
  </li>
</ul>
<hero-detail *ngIf="selectedHero" [hero]="selectedHero"></hero-detail>

Although this template uses typical HTML elements like <h2> and <p>, it also has some differences. Code like *ngFor, {{hero.name}} , (click), [hero], and <hero-detail> uses Angular's template syntax.
